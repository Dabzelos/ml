# Кластеризация Mice Protein Expression

## Описание датасета

В работе используется датасет `Mice Protein Expression` с Kaggle:

https://www.kaggle.com/datasets/ruslankl/mice-protein-expression

Датасет содержит измерения уровней экспрессии белков в коре головного мозга мышей. В таблице 1080 записей и 82 столбца. Для кластеризации используются 77 числовых признаков белков.

Важно: 1080 записей — это не 1080 разных мышей, а повторные измерения белковых профилей примерно для 72 мышей.

Мыши относятся к нескольким экспериментальным условиям:

- `Genotype`: контрольные мыши или trisomy-мыши;
- `Treatment`: инъекция `saline` или `memantine`;
- `Behavior`: стимулировались к обучению или нет;
- `class`: полная экспериментальная группа.

Всего есть 8 экспериментальных групп: `c-CS-s`, `c-CS-m`, `c-SC-s`, `c-SC-m`, `t-CS-s`, `t-CS-m`, `t-SC-s`, `t-SC-m`.

## Цель работы

Цель — решить задачу кластеризации на данных с количеством признаков больше 50.

Мы проверяем, можно ли по 77 белковым признакам автоматически выделить группы мышей и насколько найденные кластеры совпадают с известными экспериментальными факторами.

## Подготовка данных

Для кластеризации использовались только числовые белковые признаки. Служебные и категориальные столбцы `MouseID`, `Genotype`, `Treatment`, `Behavior`, `class` не использовались как признаки.

Предобработка:

- пропущенные значения заполнялись медианой по столбцу через `SimpleImputer`;
- признаки масштабировались через `StandardScaler`;
- итоговая матрица признаков имеет размер `(1080, 77)`.

## Использованные модели

В работе сравнивались три модели:

- собственная реализация `CustomKMeansPlusPlus`;
- `KMeans` из `sklearn`;
- `DBSCAN` из `sklearn`.

Собственная реализация `CustomKMeansPlusPlus` находится в файле `custom_kmeans.py`. Она реализует K-Means с инициализацией центров по алгоритму K-Means++ и несколькими запусками `n_init`, из которых выбирается решение с минимальной `inertia`.

## Подбор гиперпараметров

Для K-Means-подходов подбиралось число кластеров `k`. Для этого строились:

- график метода локтя по `inertia`;
- график `silhouette_score` для разных `k`.

В результате лучшее значение по `silhouette_score` получилось:

```text
k = 2
```

Это значение использовалось для `CustomKMeansPlusPlus` и `sklearn KMeans`.

Для DBSCAN подбирались параметры:

- `eps`;
- `min_samples`.

Дополнительно строился `k-distance plot`, чтобы выбрать разумный диапазон `eps`.

Лучший вариант DBSCAN:

```text
eps = 8
min_samples = 10
```

## Метрики качества

Для сравнения моделей использовались:

- время обучения;
- количество найденных кластеров;
- количество шумовых точек для DBSCAN;
- `inertia` для K-Means-подходов;
- `silhouette_score`;
- внутрикластерное расстояние;
- межкластерное расстояние;
- индекс Данна;
- `adjusted_rand_score`.

Для DBSCAN шумовые точки с меткой `-1` не считались отдельным кластером во внутренних метриках качества: `silhouette_score`, расстояния и индекс Данна вычислялись только по не-шумовым объектам.

`adjusted_rand_score` использовался только как дополнительная проверка: он показывает, насколько найденные кластеры совпадают с известными экспериментальными группами.

## Основные результаты

По итоговому сравнению:

```text
CustomKMeansPlusPlus -> 2 кластера
sklearn KMeans       -> 2 кластера
sklearn DBSCAN       -> 2 кластера и 23 шумовые точки
```

Лучшая модель по `silhouette_score`:

```text
sklearn DBSCAN
eps = 8
min_samples = 10
silhouette ≈ 0.370
```

При этом найденные кластеры почти не совпали напрямую с известными факторами:

- `Genotype`;
- `Treatment`;
- `Behavior`;
- `class`.

Значения `adjusted_rand_score` оказались близкими к нулю. Это означает, что найденные кластеры не являются прямым восстановлением исходных экспериментальных групп.

## PCA и визуализация

В дополнительном задании была применена PCA:

- проверялась кластеризация при разном числе главных компонент: `2, 3, 5, 10, 20, 30, 50`;
- для DBSCAN параметры `eps` и `min_samples` подбирались заново для каждого числа PCA-компонент;
- строились графики зависимости качества от числа PCA-компонент;
- данные визуализировались в 2D с помощью PCA;
- данные визуализировались в 2D с помощью t-SNE.

Для t-SNE перед запуском данные сначала сжимались через PCA до 30 компонент, чтобы ускорить расчёт.

Лучший результат после PCA показал DBSCAN при 10 главных компонентах:

```text
eps ≈ 6.097
min_samples = 10
silhouette ≈ 0.439
```

Визуализации показали, что данные имеют некоторую крупную структуру, но явного разделения на 8 исходных экспериментальных групп не наблюдается.

## Итоговый вывод

Кластеризация выявила две крупные группы в белковых профилях мышей. Однако эти группы не совпадают напрямую с генотипом, типом лечения, поведением или полными 8 экспериментальными классами.

Лучшее качество среди рассмотренных моделей показал DBSCAN, но его кластеры также не интерпретируются как прямое разделение по известным биологическим факторам.

Главный вывод: в данных есть внутренняя структура, но она сложнее, чем простое деление на `control/trisomy`, `saline/memantine` или 8 исходных классов.
